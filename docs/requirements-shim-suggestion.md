# シム提案機能 要件定義書

## 背景

現在のシム提案は通知履歴の最新データを起点にしているため、履歴を削除すると提案元を見失う問題がある。

また、実際の調整では1つのパンチだけで狙い値に入るとは限らず、フォーム、リング、逆押し、平押しの特性と、シム厚みの変更範囲を考慮した複数候補の提示が必要である。

## 目的

得意先と製品ごとの最新測定データを基準に、狙い値へ近づけるためのシム変更候補を表示する。

提案は確定指示ではなく、作業者・責任者が調整判断を行うための補助情報として扱う。

## 対象ユーザー

- 測定データを入力する作業者
- シム調整を判断する担当者
- 管理者、責任者

## 機能要件

### Must

- 得意先＋製品の最新測定データを基準にシム提案を計算する。
- 通知履歴ではなく、保存済みの測定データを提案元にする。
- 通知履歴を削除しても、測定データが残っていれば提案できる。
- 狙い値が設定されている測定項目だけを評価対象にする。
- 狙い値は範囲で扱う。
- 初期段階では、評価対象の測定項目はすべて同じ重要度で評価する。
- パンチ特性をもとに、シムを厚くした時の測定値の増減方向を判定する。
- パンチごとにシム変更範囲の上限・下限を設定できる。
- シム変更候補は1ミクロン単位で扱う。
- 複数パンチの組み合わせ候補を計算する。
- 提案候補は3件から5件を基本表示件数にする。
- 提案結果には、変更前後のシム厚み、予測測定値、狙い値との差を表示する。

### Should

- 提案候補の表示件数をユーザーが選択できる。
- 履歴ページから過去測定データを選択して「このデータで提案」できる。
- 提案候補ごとに、なぜその調整を提案したのか理由を表示する。
- 推奨変更範囲を超える候補、または上限・下限に近い候補には注意表示を出す。
- 現在のシム厚みが未入力の場合でも、変更量ベースの候補を表示できる。

### Could

- 測定項目ごとに重要度を設定できる。
- 過去データからパンチ特性を補正する。
- 製品ごとに提案候補の探索範囲、組み合わせ数、表示件数の初期値を設定できる。

### Won't

- 今回の要件では、OCRによるPDF自動読取の精度改善は対象外とする。
- 今回の要件では、外部AI APIによる予測は対象外とする。
- 今回の要件では、提案を唯一の正解として自動確定する機能は対象外とする。

## 非機能要件

- 携帯端末でも提案結果が読みやすいレスポンシブUIにする。
- 提案結果は、現場で比較しやすいよう候補単位で整理して表示する。
- 提案計算でエラーが起きた場合は、原因が分かるメッセージを表示する。
- 認証済みユーザーのみシム提案を利用できる。
- 得意先・製品に紐づかないデータを提案に混ぜない。
- Supabase上の既存測定データを破壊しない。

## 未確定事項

- 過去データを使った提案はShould要件とし、最初の実装範囲に含めるかは設計フェーズで判断する。
- 提案候補の最大組み合わせパンチ数は未確定。初期案は最大2パンチ。
- 提案候補の表示件数選択は、3件または5件を選べる形を初期案とする。
- パンチごとのシム変更範囲設定をどの画面に配置するかは未確定。
- 重要度設定はCould要件とし、初回実装では同一評価とする。

## 次フェーズへの引き継ぎ事項

- Designフェーズでは、通知履歴ではなくmeasurement_setsを提案元にするAPI設計へ変更する。
- Designフェーズでは、パンチごとのシム変更範囲を保存するDB設計を検討する。
- Designフェーズでは、最新測定データ提案と過去データ選択提案を同時に実装するか、段階実装にするかを整理する。
- Designフェーズでは、シム提案の計算ロジック、候補数、組み合わせパンチ数、UI表示項目を具体化する。
